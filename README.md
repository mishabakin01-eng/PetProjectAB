A/B ТЕСТИРОВАНИЕ ПОЛОЖЕНИЯ ГЕЙТА В МОБИЛЬНОЙ ИГРЕ COOKIE CATS
================================================================================

1. ОПИСАНИЕ ПРОЕКТА
--------------------------------------------------------------------------------
Данный проект представляет собой анализ результатов A/B теста мобильной игры 
Cookie Cats. Цель теста - определить оптимальное положение "гейта" (точки, 
где игроку предлагается заплатить или подождать): на 30-м или на 40-м уровне.

Основная бизнес-задача: выявить версию, которая максимизирует удержание 
игроков (retention) и потенциальную выручку, не ухудшая общий игровой опыт.

2. ОПИСАНИЕ ДАТАСЕТА
--------------------------------------------------------------------------------
Файл: cookie_cats.txt
Строки: ~90,000 пользователей
Столбцы:
  - userid         : Уникальный идентификатор игрока
  - version        : Версия игры ('gate_30' или 'gate_40')
  - sum_gamerounds : Общее количество пройденных раундов
  - retention_1    : Вернулся ли игрок на следующий день (True/False)
  - retention_7    : Вернулся ли игрок через 7 дней (True/False)

Дополнительно введена кастомная метрика:
  - gt_pass        : Количество пройденных гейтов (sum_gamerounds / номер уровня с гейтом).
                     Является метрико для оценки потенциальной выручки.

3. ГИПОТЕЗЫ
--------------------------------------------------------------------------------
H0 (Нулевая): Смена положения гейта не влияет на ключевые метрики.
H1 (Альтернативная): Версия 'gate_30' увеличивает метрики retention_1, 
     retention_7, mean_gamerounds и mean_gt_pass более чем на 1%.

4. МЕТОДОЛОГИЯ И СТАТИСТИЧЕСКИЙ ДИЗАЙН
--------------------------------------------------------------------------------
Инструменты: Python, Pandas, NumPy, SciPy, Matplotlib.

Параметры расчета размера выборки:
  - MDE (Минимальный детектируемый эффект) : 1% (0.01)
  - Alpha (Уровень значимости)             : 0.05
  - Power (Статистическая мощность)        : 90% (0.90)

Проверка объема выборки подтвердила, что имеющихся данных (~45,000 на группу) 
достаточно для обнаружения заданного эффекта для метрик retention.

Для проверки гипотез использовались:
  - Z-тест для двух пропорций (для retention_1 и retention_7)
  - T-тест Уэлча для сравнения средних (для gamerounds и gt_pass)
  - Расчет 95% доверительных интервалов (Confidence Intervals)

6. ВЫВОДЫ И РЕКОМЕНДАЦИИ
--------------------------------------------------------------------------------
1. ОСНОВНОЙ ВЫВОД: Версия gate_30 показывает статистически значимое 
   превосходство по метрикам retention_7 и mean_gt_pass.

2. МОНЕТИЗАЦИЯ: Значимый рост метрики mean_gt_pass указывает на то, что 
   игроки в версии gate_30 проходят больше монетизационных точек, что 
   напрямую связано с потенциальной прибылью компании.

3. РЕКОМЕНДАЦИЯ:
    -Рекомендуется выпустить и масштабировать версию gate_30 
   на 100% трафика.
   -  Проанализировать влияние на выручку.

4. СЛЕДУЮЩИЕ ШАГИ:
   - Гипотеза об увеличении retention_1 и mean_gamerounds не подтвердилась 
     на текущем уровне статистической значимости.
   - Для получения однозначных выводов по этим метрикам необходимо либо 
     увеличить объем выборки.
